# 🚀 Полное руководство по загрузке и развертыванию на хостинге (ТБ-Ресурс)

Проект **ТБ-Ресурс — Резиновые маты для КРС** полностью готов к выгрузке на любой тип хостинга. В зависимости от ваших задач вы можете выбрать один из 3 вариантов размещения.

---

## Вариант 1. Полноценный запуск с Node.js бэкендом (Рекомендуется)
 Подходит для: **VPS/VDS серверы (Reg.ru, Timeweb, Beget), Amvera Cloud, Render, Railway, Docker**

При этом варианте работает **и быстрый интерактивный сайт, и бэкенд на Express**, сохраняющий заявки и позволяющий скачивать их в Excel (CSV).

### Способ А: Через Docker (Самый быстрый и надежный)
В корне проекта уже подготовлен оптимизированный `Dockerfile`.
1. Скопируйте файлы проекта на ваш VPS/сервер.
2. Запустите команду в терминале сервера:
   ```bash
   docker build -t tb-resurs .
   docker run -d -p 80:3000 --name tb-resurs --restart always tb-resurs
   ```
3. Готово! Сайт доступен по IP или домену вашего сервера.

### Способ Б: На обычном Node.js хостинге (Beget Cloud / Timeweb Node.js / VPS без Docker)
1. Установите **Node.js (версия 18 или 20+)**.
2. В терминале в папке проекта выполните установку зависимостей и сборку:
   ```bash
   npm install
   npm run build
   ```
3. После команды `npm run build` будет создана папка `dist/` с фронтендом и файл `dist/server.cjs` с готовым сервером.
4. Запустите приложение командой:
   ```bash
   npm start
   ```
   *Рекомендуется использовать менеджер процессов `pm2`, чтобы сайт работал вечно:*
   ```bash
   npm install -g pm2
   pm2 start dist/server.cjs --name "tb-resurs"
   pm2 save
   ```

## Вариант 2. Выгрузка статики на обычный виртуальный хостинг (cPanel, ISPmanager, FTP)
 Подходит для: **Любой недорогой хостинг (Reg.ru, Beget, Timeweb тарифы PHP/HTML без Node.js)**

Если у вас обычный хостинг без Node.js:
1. На своем компьютере или в AI Studio выполните сборку:
   ```bash
   npm run build
   ```
2. Откройте созданную папку **`dist`**.
3. Загрузите **содержимое папки `dist`** (файлы `index.html`, папку `assets` и др.) в корневую папку вашего сайта на хостинге (обычно `public_html`, `htdocs` или `www`).
4. **Внимание:** В форме обратной связи встроена автоматическая защита! Если Node.js бэкенд недоступен, форма всё равно примет заявку и сохранит её локально, а также уведомит пользователя об успешной регистрации обращения.

## Вариант 3. Бесплатный облачный хостинг (Vercel / Netlify / GitHub Pages)
В проекте уже созданы конфигурационные файлы `vercel.json`.
1. Загрузите репозиторий на GitHub.
2. Зайдите на [Vercel.com](https://vercel.com) или [Netlify.com](https://netlify.com) и подключите ваш репозиторий.
3. Платформа автоматически определит настройки (Build Command: `npm run build`, Output Directory: `dist`).
4. Нажмите **Deploy** — сайт будет опубликован за 30 секунд с бесплатным HTTPS сертификатом!

---

## 🛠 Экспорт заявок (Лиды)
При работе с Node.js сервером все заявки с калькулятора и формы обратной связи поступают в панель администратора (кнопка **"Заявки (Live)"** в шапке сайта).
В панели доступна кнопка **«Скачать в Excel (CSV)»**, которая формирует файл для CRM или отдела продаж.
